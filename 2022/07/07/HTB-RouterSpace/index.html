HTB-RouterSpace

发表于2022-07-07|更新于2022-07-07|靶场Hacktheboxeasy
|字数总计:1.1k|阅读时长:5分钟|阅读量:|评论数:

信息搜集

nmap -A -sC -sV -T4 10.10.11.148

开启了SSH服务和80Web服务，网页端可以下载一个apk安装文件，先测试一下安卓客户端。

安卓程序测试

为了在Linux主机上运行安卓程序，需要一个模拟软件，这里使用Anbox。
关于Anbox的介绍和使用方法可以参考这个链接，英文完整教程安装+运行+代理设置。

snap run anbox launch --package=org.anbox.appmgr --component=org.anbox.appmgr.AppViewActivity 命令行运行anbox模拟器
adb devices 启动adb服务并查看anbox是否成功开启
adb install RouterSpace.apk 安装下载的apk软件

安装成功后会出现该软件的图标：

Anbox代理设置

ifconfig anbox0 查看Anbox默认的网桥
adb shell settings put global http_proxy 192.168.250.1:6666 通过adb设置Anbox的全局代理IP和端口
adb shell settings put global http_proxy :0 取消代理

设置好代理后使burp suite监听该地址即可，按上面配置，地址应该设置为192.168.250.1:6666，如下图：

抓到的数据包：

发现是向routerspace.htb域名发送数据，首先将该域名添加到/etc/hosts文件里面。


发现存在命令执行漏洞，接下来获取shell。

普通权限

发现存在命令执行后，我们最终的目的是获取到一个ssh连接。

通过echo和重定向命令测试是否可以成功存储公钥。

发现可以成功存储，接下来将公钥写入.ssh目录下面。

ssh paul@10.10.11.148 -i ./id_rsa 使用私钥登录服务器即可

root权限

find / -perm /4000 2>/dev/null 查看具有SUID权限的文件

没有发现什么特别的文件，接下来使用提权脚本跑一下。

使用curl和wget均无法下载文件，但可以使用scp通过ssh传输文件：scp -i ./id_rsa ./local_file paul@10.10.11.148:/remote_dir
使用LES脚本来发现提权点，用同样的方法上传该脚本文件。

发现CVE-2021-3156，接下来使用脚本利用。
tar -cvf exp.tar CVE-2021-3156打包整个文件夹后发送到靶机上。

成功的获取到了root权限。
